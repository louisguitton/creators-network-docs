OneFootball Network API¶

Einführung¶

OneFootball ist die weltweite Nummer 1 unter den digitalen Fußball-Plattformen. Die OneFootball Network API erlaubt es den Content-Partnern von OneFootball, Artikel auf der Plattform von OneFootball zu veröffentlichen. Zudem können unsere Partner Artikel, die sie auf der OneFootball-Plattform veröffentlicht haben, jederzeit aktualisieren oder löschen.
Unsere Partner können die API nutzen, um ihre eigenen Services einzurichten. So kann zum Beispiel Content, der auf der Klubseite veröffentlich wird, automatisch auch an OneFootball geschickt und nativ auf der OneFootball-Plattform für die User von OneFootball integriert werden.
Das OneFootball Network stellt eine geschlossene Plattform dar, auf der man nur auf Einladung arbeiten kann. Nur autorisierte Content-Partner sind in der Lage, die OneFootball Network API zu nutzen.

Richtlinien¶

Bevor Ihr beginnt, beachtet bitte die folgenden Richtlinien, die für die gesamte API gelten.

Schema¶

Zugang zur API gibt es nur über HTTPS. Alle Daten werden im JSON-Format versendet und empfangen.
Zugang gibt es über folgende URLs: https://network-api.onefootball.com

Zeitformat¶

Alle Zeitstempel sind im ISO 8601-Format YYYY-MM-DDTHH:MM:SSZ.

Authentifizierung¶

Anfragen, die eine Authentifizierung benötigen, müssen den Header Authorization haben. Der Wert muss dem Bearer Token entsprechen, den man während des LogIn-Prozesses angefordert hat.

Beispiele¶

Statt das Vorgehen in Worten zu beschreiben, haben wir in der folgenden Dokumentation versucht, Beispiele der Code Snippets zu zeigen, die man kopieren, einfügen und anpassen kann. Bitte seht euch die Beispiele als Referenz an.

Testing¶

Wenn Ihr die Veröffentlichung testen und Artikel aktualisieren möchtet, müsst Ihr die optionale Boolean draft nutzen. Wenn Ihr eine Anfrage mit draft ist TRUE sendet, ist sichergestellt, dass der jeweilige Artikel für die User von OneFootball nicht sichtbar ist.
Wenn Ihr soweit seid, eure Services im Produktionsprozess zu nutzen und Artikel zu schicken, die für die User von OneFootball sichtbar sind, könnt Ihr entweder den Parameter draft auslassen oder von draft auf FALSE abändern.

Set-Up¶

Wenn Ihr Content auf OneFootball über die OneFootball Network API veröffentlichen, updaten oder löschen möchtet, müsst Ihr zunächst Folgendes unternehmen:

  1. Schließt die Registrierung für euren OneFootball Network-Account ab.

  2. Benutzt eure Anmeldedaten, um einen Authentifizierungs-Token anzufordern, den Ihr dann für weitere Anforderungen nutzen könnt.

  3. Fordert dann die ID der Integration an, unter der Ihr euren Content veröffentlichen wollt (ACHTUNG: viele Accounts haben nur eine Webseite und damit nur eine Integration).

Aktiviert euren OneFootball Network-Account¶

Zur Plattform des OneFootball Networks erhaltet Ihr nur auf Einladung Zugang. Wenn Ihr eingeladen seid, und zugestimmt habt, erhaltet Ihr eine Registrierungs-E-Mail. Bitte befolgt die Instruktionen in dieser E-Mail, erstellt euer Password und loggt euch dann in das Portal des OneFootball Networks ein. Ihr müsst diesen Onboarding-Prozess im Portal abgeschlossen haben, bevor Ihr euren Content über die OneFootball Network API veröffentlichen könnt.

Authentifizierungs-Token anfordern¶

Sobald Ihr euch erfolgreich ins Portal des OneFootball Networks eingeloggt habt , könnt Ihr mit derselben E-Mail-Adresse und demselben Password einen Authentifizierungs-Token für die OneFootball Network API anfordern.
Um euren Content über die OneFootball Network API zu publishen, aktualisieren oder löschen, müsst Ihr einen gültigen Authentifizierungs-Token haben. Um einen Token zu erhalten, müsst Ihr eure Anmeldedaten wie im folgenden Beispiel senden:
$ curl -X POST \
    https://network-api.onefootball.com/v1/login \
    -H "Content-Type: application/json" \
    -d '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

Authentifizierungs-Token erneuern¶

Jeder Authentifizierungs-Token ist nach erstmaligem Gebrauch sieben Tage gültig. Sobald euer Token abgelaufen ist, könnt Ihr - wie oben beschrieben - mit euren Anmeldedaten einen neuen anfordern.

Alle registrierten Integrations auflisten¶

Um euren Content zu veröffentlichen, müsst Ihr präzisieren, zu welchen eurer Integrations der Content gehört. In den meisten Fällen wird euer Account nur eine Integration mit OneFootball haben. Accounts, mit denen Content von verschiedenen Webseiten auf OneFootball veröffentlicht werden, können allerdings mehrere Integrations haben und müssen daher die korrekte Integration für die richtige Webseite auswählen.
Ihr könnt eine komplette Liste all eurer registrierten Integrations auf OneFootball erhalten, indem Ihr diesem Beispiel folgt:

  • Ersetzt TOKEN im Header durch euren gültigen Authentifizierungs-Token.

$ curl -X GET \
    https://network-api.onefootball.com/v1/integrations/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

headers = {

Content veröffentlichen¶

Wenn euer Set-Up abgeschlossen ist und Ihr im Besitz eines Authentifizierungs-Tokens seid, könnt Ihr damit beginnen, Artikel auf OneFootball zu veröffentlichen.

Einen Artikel publishen¶

Mit einem gültigen Authentifizierungs-Token könnt Ihr einen Artikel veröffentlichen. Sendet dazu die Aufforderung POST an den Entity-Endpunkt des Posts.
Wie das geht, zeigt das folgende Beispiel:

  • Ersetzt TOKEN im Header durch euren gültigen Authentifizierungs-Token.

  • Setzt alle Attribute eures Artikels wie im folgenden Beispiel.

$ curl -X POST \
    https://network-api.onefootball.com/v1/posts/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language": "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",
        "content": "Article content",
        "title": "Article title",
        "image_url": "https://your-blog.com/images/1.png",
        "image_width": 200,
        "image_height": 100,
        "draft": false
    }'

Content aktualisieren oder löschen¶

Sobald euer Content auf OneFootball ist, könnt Ihr ihn jederzeit updaten oder wieder löschen.
Um einen Artikel auf OneFootball zu bearbeiten oder zu löschen, benötigt Ihr zuerst eine OneFootball Post ID für diesen bestimmten Artikel.

Post ID des Artikels¶

Um die OneFootball Post ID für einen Artikel zu erhalten, den Ihr updaten oder löschen möchtet, folgt diesem Beispiel:

  • Ersetzt EXTERNAL_ID in der URL mit der ID des Artikels in eurem System, die Ihr generiert habt, als Ihr den Artikel veröffentlicht habt. Falls nötig, gerne auch verschlüsselt.

  • Ersetzt TOKEN im Header durch euren gültigen Authentifizierungs-Token.

$ curl -X GET \
    https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

import requests

headers = {
    'Authorization': 'Bearer TOKEN',
}

params = (
    ('external_id', 'EXTERNAL_ID'),
)

response = requests.get('https://network-api.onefootball.com/v1/posts/', headers=headers, params=params)

Einen Artikel aktualisieren¶

Ab und an werdet Ihr einen Artikel von eurer Seite aktualisieren müssen. Um sicher zu stellen, dass dieser Artikel auch auf OneFootball aktualisiert wird, nutzt die PUT Methode des Entity-Endpunktes des Posts.
Wie das geht, zeigt das folgende Beispiel:

  • Ersetzt POST_ID in der URL durch die OneFootball Post ID des Artikels, den Ihr aktualisieren möchtet (wie das geht, haben wir weiter oben beschrieben).

  • Ersetzt TOKEN im Header durch euren gültigen Authentifizierungs-Token.

  • Setzt alle Attribute eures Artikels wie im folgenden Beispiel. ACHTUNG: alle Attribute überschreiben existierende Werte.

$ curl -X PUT \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language":  "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",
        "content":  "Article content",
        "title":  "Article title",
        "image_url":  "https://your-blog.com/images/1.png",

        "draft": false
