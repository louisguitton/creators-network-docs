API da OneFootball Network¶

Introdução¶

OneFootball é a maior plataforma digital de futebol do mundo. A API da OneFootball Network permite que os parceiros de conteúdo do OneFootball publiquem notícias diretamente na plataforma OneFootball. Ela também permite que os parceiros atualizem e excluam artigos que já foram publicados na plataforma OneFootball.
A API pode ser usada pelos parceiros para configurar os seus próprios serviços, de modo que o conteúdo publicado nos seus sites seja automaticamente enviado para o OneFootball e consequentemente seja lido pelos utilizadores do OneFootball.
É importante destacar que a OneFootball Network é uma plataforma fechada, acessível apenas para convidados, e somente parceiros de conteúdo autorizados podem usar a API da OneFootball Network.

Convenções¶

Antes de começar, observe as convenções listadas aqui, que se aplicam a toda a API.

Esquema¶

Todo o acesso à API será feito por HTTPS. Todos os dados são enviados e recebidos em formato JSON.
O acesso pode ser feito através do seguinte URL: https://network-api.onefootball.com

Formato da data¶

Todas a s datas estão no formato ISO 8601 YYYY-MM-DDTHH:MM:SSZ.

Autenticação¶

Requests que exigem autenticação precisam fornecer um cabeçalho Authorization. O valor deve ser um token de verificação Bearer adquirido durante o processo de login.

Exemplos¶

Na documentação, tentamos fornecer exemplos de código que podem ser copiados, colados e ajustados, em vez de descrever tudo em palavras. Por favor, use estes exemplos como referência.

Teste¶

Ao testar a publicação e atualização de notícias, use o draft booleano opcional. Um request enviado como draft a TRUE garantirá que a respectiva notícia não fique visível para os utilizadores do OneFootball.
Quando você estiver pronto para usar o seu serviço em produção e enviar notícias reais para serem vistas pelos utilizadores do OneFootball, você pode omitir o parâmetro draft ou definir o draft como FALSE.

Como configurar¶

Para publicar, atualizar ou excluir conteúdo do OneFootball através da API, você deve primeiramente fazer o seguinte:

  1. Registe a sua conta da OneFootball Network.

  2. Use os seus dados de login para obter um token de autenticação que deve ser usado para futuros requests.

  3. Obtenha o ID da integração na qual você deseja publicar o conteúdo (observe que muitas contas terão apenas um site e, portanto, apenas uma integração).

Ativando sua conta da OneFootball Network¶

A OneFootball Network é uma plataforma apenas para convidados. Se você for convidado e concordou em participar, receberá um e-mail de registo. Siga as instruções do e-mail de registo, defina a sua senha e faça o login no portal da OneFootball Network. Lembre-se de concluir o processo de registo no portal antes de tentar publicar conteúdo através da API da OneFootball Network.

Obter um token de autenticação¶

Depois de fazer o login no portal da OneFootball Network, você pode usar o mesmo endereço de e-mail e senha para obter um token de autenticação na API da OneFootball Network.
Você precisa ter um token de autenticação válido para publicar, alterar ou excluir conteúdo através da API da OneFootball Network. Para adquirir um token, envie seus dados de login conforme o exemplo abaixo:
$ curl -X POST \
    https://network-api.onefootball.com/v1/login \
    -H "Content-Type: application/json" \
    -d '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

import requests

headers = {
    'Content-Type': 'application/json',
}

data = '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

response = requests.post('https://network-api.onefootball.com/v1/login', headers=headers, data=data)

Atualizando um token de autenticação¶

Cada token de autenticação é válido por sete dias após a emissão. Quando um token expirar, repita o processo usando os seus dados de login para adquirir um novo token.

Listando todas as integrações registadas¶

Para publicar conteúdo, você deve especificar qual a integração a que o conteúdo pertence. Na maioria dos casos, a sua conta terá apenas uma integração com o OneFootball. No entanto, contas com vários sites diferentes que publicam conteúdo no OneFootball podem precisar realizar diversas integrações e devem, portanto, selecionar a integração certa para o site certo.
Você pode obter a lista completa das suas integrações seguindo o exemplo abaixo e fazendo o seguinte:

  • Substitua TOKEN no cabeçalho pelo seu token de autenticação válido.

$ curl -X GET \
    https://network-api.onefootball.com/v1/integrations/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

Publicando conteúdo¶

Depois de configurar e ter um token de autenticação, você pode publicar uma noticía no OneFootball.

Publicando uma notícia¶

Usando um token de autenticação válido, você pode publicar uma notícia enviando um request POST.
Para isso, siga o exemplo abaixo e faça o seguinte:

  • Substitua TOKEN no cabeçalho pelo seu token de autenticação válido.

  • Defina todos os atributos da notícia como mostrado no exemplo abaixo.

$ curl -X POST \
    https://network-api.onefootball.com/v1/posts/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language": "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",
        "content": "Article content",
        "title": "Article title",
        "image_url": "https://your-blog.com/images/1.png",
        "image_width": 200,
        "image_height": 100,
        "draft": false
    }'

Atualizando e excluindo conteúdo¶

Assim que o seu conteúdo for publicado no OneFootball, é possível atualizá-lo e excluí-lo.
Para atualizar ou excluir uma notícia no OneFootball, você precisará de obter o ID da mesma.

Obtendo um ID de uma notícia¶

Para obter o ID de uma notícia no OneFootball siga o exemplo abaixo e faça o seguinte:

  • Substitua o EXTERNAL_ID pelo ID da notícia no seu sistema, que você forneceu ao publicar a notícia. Isto deve ser codificado se necessário.

  • Substitua TOKEN no cabeçalho pelo seu token de autenticação válido.

$ curl -X GET \
    https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

import requests

headers = {
    'Authorization': 'Bearer TOKEN',
}

params = (
    ('external_id', 'EXTERNAL_ID'),
)

response = requests.get('https://network-api.onefootball.com/v1/posts/', headers=headers, params=params)

req, err := http.NewRequest("GET", "https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID", nil)
if err != nil {
 // handle err
}

req.Header.Set("Authorization", "Bearer TOKEN")

resp, err := http.DefaultClient.Do(req)

defer resp.Body.Close()

Atualizando uma notícia¶

Em certas ocasiões, você poderá editar uma notícia. Para garantir que esta é atualizada no OneFootball, use o método PUT.
Para isso, siga o exemplo abaixo e faça o seguinte:

  • Substitua POST_ID no URL pelo ID da notícia que você deseja atualizar (veja acima como obtê-lo).

  • Substitua TOKEN no cabeçalho pelo seu token de autenticação válido.

  • Defina todos os atributos da notícia como mostrado no exemplo abaixo. Observe que todos os atributos substituirão os valores existentes.

$ curl -X PUT \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language":  "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",

Excluindo uma notícia¶

Você também pode excluir notícias do OneFootball usando o método DELETE.
Para isso, siga o exemplo abaixo e faça o seguinte:

  • Substitua POST_ID pelo ID da notícia que você deseja apagar (veja acima como obtê-lo).

  • Substitua TOKEN no cabeçalho pelo seu token de autenticação válido.

$ curl -X DELETE \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

import requests

headers = {
    'Authorization': 'Bearer TOKEN',
}

response = requests.delete('https://network-api.onefootball.com/v1/posts/POST_ID', headers=headers)

req.Header.Set("Authorization", "Bearer TOKEN")
