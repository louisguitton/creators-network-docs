API OneFootball Network¶

Introduction¶

OneFootball est la plus grande plateforme digitale au monde entièrement consacrée au football. L’API du OneFootball Network permet aux partenaires de OneFootball de publier des articles sur la plateforme. Il leur permet également de mettre à jour et de supprimer des articles qu’ils ont déjà publiés.
Nos partenaires peuvent utiliser l’interface de programmation (API) pour mettre en place leurs services, afin que les contenus publiés sur leurs sites respectifs soient automatiquement envoyés sur OneFootball, intégrés à sa plateforme et consultables par ses utilisateurs.
Veuillez noter que le OneFootball Network est un réseau fermé, accessible seulement sur invitation, et que seuls les partenaires autorisés peuvent avoir accès à l’API de OneFootball.

Utilisation¶

Avant de débuter, veuillez noter les normes d’utilisation listées ci-dessous et valables à travers l’API.

Schéma¶

Tout accès API se fait via HTTPS. Toutes les données sont envoyées et reçues en format JSON.
L’accès se fait via les URL suivants : https://network-api.onefootball.com

Format temporel¶

Tous les horodatages reviennent en ISO 8601 format YYYY-MM-DDTHH:MM:SSZ.

Authentification¶

Les demandes d’authentification doivent comporter l’en-tête Authorization. La valeur doit être un token Bearer obtenu au cours du processus d’identification.

Exemples¶

Dans cette documentation, nous donnons des exemples de code pouvant être copié, collé, et ajusté, plutôt que de tout expliquer à l’aide de mots. Veuillez vous servir de ces exemples comme référence.

Testing¶

Pour tester les publications et mettre à jour les articles, utilisez la booléenne optionnelle draft. Une demande envoyée avec draft qui est TRUE fera en sorte que l’article ne sera pas visible par les utilisateurs OneFootball.
Une fois prêts à produire et envoyer de vrais articles accessibles aux utilisateurs de OneFootball, vous pouvez soit écarter le paramètre draft, soit paramétrer draft en FALSE.

Mise en place¶

Afin de publier, mettre à jour ou supprimer du contenu sur OneFootball via son API, vous devez d’abord :

  1. Compléter l’inscription de votre compte sur le OneFootball Network.

  2. Utiliser vos identifiants de connexion pour obtenir un token d’authentification, lequel devra être utilisé pour d’autres demandes.

  3. Obtenir l’identifiant d’intégration sous lequel vous souhaitez publier du contenu (veuillez noter que de nombreux comptes ne comporteront qu’un seul site internet, et donc une seule intégration).

Activer votre compte sur le OneFootball Network¶

Le OneFootball Network est accessible uniquement sur invitation. Si vous acceptez l’invitation, vous recevrez un email d’inscription. Veuillez suivre les instructions énoncées dans cet email, choisissez un mot de passe, puis connectez-vous au portail du OneFootball Network. Assurez-vous de compléter le processus d’intégration sur le portail avant de publier du contenu via l’API du OneFootball Network.

Obtenir un token d’authentification¶

Une fois connecté au portail du OneFootball Network, vous pouvez utiliser la même adresse email et le même mot de passe pour obtenir un token d’authentification pour l’API du OneFootball Network.
Vous devez disposer d’un token d’authentification valide afin de publier, modifier ou supprimer du contenu via des demandes envoyées à l’API du OneFootball Network. Pour obtenir un token, envoyez vos identifiants de connexion comme dans l’exemple ci-dessous :
$ curl -X POST \
    https://network-api.onefootball.com/v1/login \
    -H "Content-Type: application/json" \
    -d '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

import requests

headers = {
    'Content-Type': 'application/json',
}

data = '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

response = requests.post('https://network-api.onefootball.com/v1/login', headers=headers, data=data)

 Login    string `json:"login"`

}

// fill struct
}
payloadBytes, err := json.Marshal(data)
if err != nil {

Rafraîchir un token d’authentification¶

Chaque token d’authentification est valide pendant sept jours. Une fois qu’un token arrive à expiration, répétez le processus avec vos identifiants de connexion pour en obtenir un autre.

Lister toutes les intégrations enregistrées¶

Afin de publier du contenu, vous devez spécifier à laquelle de vos intégrations ledit contenu appartient. Dans la plupart des cas, votre compte n’aura qu’une intégration à OneFootball. Néanmoins, les comptes ayant plusieurs sites internet différents publiant sur OneFootball peuvent disposer de plusieurs intégrations, et doivent ainsi sélectionner la bonne intégration correspondant au bon site internet.
Vous pouvez obtenir une liste complète de vos intégrations sur OneFootball en vous référant à l’exemple ci-dessous :

  • Remplacez TOKEN dans l’en-tête par votre token d’authentification valide.

    https://network-api.onefootball.com/v1/integrations/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

    'Authorization': 'Bearer TOKEN',
}

Publier du contenu¶

Une fois la mise en place effectuée et le token d’authentification obtenu, vous pouvez publier un article sur OneFootball.

Publier un article¶

Avec un token d’authentification valide, vous pouvez publier un article en envoyant une demande POST à l’endpoint posts.
Pour ce faire, veuillez vous référer à l’exemple ci-dessous :

  • Remplacez TOKEN dans l’en-tête par votre token d’authentification valide.

  • Remplacez TOKEN dans l’en-tête par votre token d’authen’ valide.

$ curl -X POST \
    https://network-api.onefootball.com/v1/posts/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \

        "external_id": "ARTICLE_ID",

Modifier et supprimer du contenu¶

Une fois que votre contenu est présent sur OneFootball, vous pouvez le mettre à jour et le supprimer.
Pour ce faire, vous devez vous munir de l’ID OneFootball de la publication en question.

Obtenir l’ID de publication¶

Pour obtenir l’ID OneFootball de publication d’un article pour sa modification/suppression, veuillez vous référer à l’exemple ci-dessous :

  • Remplacez EXTERNAL_ID dans l’URL avec l’ID de l’article dans votre système que vous avez fourni au moment de la publication de l’article ; à encoder si nécessaire.

  • Remplacez TOKEN dans l’en-tête par votre token d’authentification valide.

$ curl -X GET \
    https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

import requests

headers = {
    'Authorization': 'Bearer TOKEN',
}

params = (
    ('external_id', 'EXTERNAL_ID'),
)

response = requests.get('https://network-api.onefootball.com/v1/posts/', headers=headers, params=params)

req, err := http.NewRequest("GET", "https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID", nil)
if err != nil {
 // handle err
}
req.Header.Set("Content-Type", "application/json")
req.Header.Set("Authorization", "Bearer TOKEN")

resp, err := http.DefaultClient.Do(req)
if err != nil {

defer resp.Body.Close()

Modifier un article¶

Occasionnellement, vous pouvez être amené à modifier un article sur votre site internet. Afin que cette modification apparaisse sur OneFootball, utilisez la méthode PUT de l’endpoint posts.
Pour ce faire, veuillez vous référer à l’exemple ci-dessous :

  • Remplacez POST_ID dans l’URL par l’ID de publication OneFootball de l’article que vous souhaitez modifier (pour obtenir cet ID, voir ci-dessus).

  • Remplacez TOKEN dans l’en-tête par votre token d’authentification valide.

  • Paramétrez votre article tel que dans l’exemple ci-dessous. Veuillez noter que votre paramétrage remplacera les paramètres existants.

$ curl -X PUT \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language":  "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",
        "content":  "Article content",
        "title":  "Article title",

Supprimer un article¶

Vous pouvez aussi supprimer des articles de OneFootball en utilisant la méthode DELETE de l’endpoint posts.
Pour ce faire, veuillez vous référer à l’exemple ci-dessous :

  • Remplacez POST_ID dans l’URL par l’ID OneFootball de publication de l’article que vous souhaitez supprimer (pour obtenir cet ID, voir ci-dessus)

  • Remplacez TOKEN dans l’en-tête par votre token d’authentification valide.

$ curl -X DELETE \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

import requests

headers = {
    'Authorization': 'Bearer TOKEN',
}

response = requests.delete('https://network-api.onefootball.com/v1/posts/POST_ID', headers=headers)

req, err := http.NewRequest("DELETE", "https://network-api.onefootball.com/v1/posts/POST_ID", nil)
if err != nil {
 // handle err

req.Header.Set("Content-Type", "application/json")
req.Header.Set("Authorization", "Bearer TOKEN")

resp, err := http.DefaultClient.Do(req)
