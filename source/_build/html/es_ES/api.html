OneFootball Network API¶

Introducción¶

OneFootball es la plataforma de fútbol digital líder en el mundo. La API de la OneFootball Network permite a los socios de contenido publicar artículos en la propia plataforma. También permite a los socios actualizar y eliminar artículos que Ya han publicado en la plataforma.
Los socios pueden utilizar la API para configurar sus propios servicios de modo que el contenido publicado en sus sitios se envíe automáticamente a OneFootball y se integre de forma nativa en la plataforma OneFootball para los usuarios de OneFootball.
Por favor tenga en cuenta que la OneFootball Network es una plataforma cerrada, solo para invitados, y solo los socios de contenido autorizados pueden usar la API de la red OneFootball.

Convenciones¶

Antes de empezar, tenga en cuenta las convenciones aquí enumeradas, las cuales se aplican en toda la API.

Esquema¶

Todo el acceso a la API se realiza a través de HTTPS. Todos los datos se envían y reciben en formato JSON.
El acceso es desde la siguiente URL: https://network-api.onefootball.com

Formato de tiempo¶

Todas las marcas de tiempo están en formato ISO 8601 YYYY-MM-DDTHH:MM:SSZ.

Autentificación¶

Las solicitudes que requieren autenticación deben proporcionar un encabezado Authorization. El valor debe ser un token Bearer adquirido durante el proceso de inicio de sesión.

Ejemplos¶

En la documentación, hemos tratado de dar ejemplos de fragmentos de código que se pueden copiar, pegar y ajustar en lugar de describir todo en palabras. Por favor, mire los ejemplos como referencia.

Prueba¶

Al probar la publicación y actualización de artículos, utilice el draft booleano opcional. Una solicitud enviada con draft como TRUE garantizará que el artículo respectivo no sea visible para los usuarios de OneFootball.
Cuando esté listo para usar su servicio en producción y enviar artículos reales para que los vean los usuarios de OneFootball, puede omitir el parámetro draft o establecer draft como FALSE.

Cómo configurar¶

Para publicar, actualizar o eliminar contenido en OneFootball a través de la API de la red OneFootball, primero debe hacer lo siguiente:

  1. Complete el registro para su cuenta en la OneFootball Network.

  2. Use sus credenciales de inicio de sesión para obtener un token de autentificación que debe usarse para otras solicitudes.

  3. Obtenga la ID de la integración en la que desea publicar contenido (tenga en consideración que muchas cuentas tendrán un solo sitio web y, por lo tanto, solo una integración).

Activación de su cuenta en la OneFootball Network¶

La OneFootball Network en una plataforma solo por invitación. Si está invitado a unirse y ha aceptado unirse, recibirá un correo electrónico de registro. Siga las instrucciones en el correo electrónico de registro, configure su contraseña y luego inicie sesión en el portal de la OneFootball Network. Asegúrese de completar el proceso de incorporación en el portal antes de intentar publicar contenido a través de la API de la OneFootball Network.

Obtener el token de autentificación¶

Una vez que haya iniciado sesión con éxito en el portal de la OneFootball Network, puede usar la misma dirección de correo electrónico y contraseña para obtener un token de autenticación para la API de la red OneFootball.
Debe tener un token de autenticación válido para publicar, modificar o eliminar contenido a través de solicitudes realizadas a la API de la OneFootball Network. Para adquirir un token, envíe sus credenciales de inicio de sesión según el siguiente ejemplo:
$ curl -X POST \
    https://network-api.onefootball.com/v1/login \
    -H "Content-Type: application/json" \
    -d '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

import requests

headers = {

}

data = '{"login": "EMAIL_ADDRESS", "password": "SECRET_PASSWORD"}'

response = requests.post('https://network-api.onefootball.com/v1/login', headers=headers, data=data)

Actualizar un token de autentificación¶

Cada token de autenticación es válido durante siete días después de su emisión. Después de que caduque un token, repita el proceso utilizando sus credenciales de inicio de sesión para adquirir uno nuevo.

Listado de todas las integraciones registradas¶

Para publicar contenido, debe especificar a qué integraciones pertenece el contenido. En la mayoría de los casos, su cuenta solo tendrá una integración con OneFootball. Sin embargo, las cuentas con múltiples sitios web diferentes que publican contenido en OneFootball pueden tener múltiples integraciones y, por lo tanto, hay que seleccionar la integración correcta para el sitio web correcto.
Puede obtener una lista completa de sus integraciones registradas en OneFootball tomando el siguiente ejemplo y haciendo lo siguiente:

  • Reemplace TOKEN en el encabezado con su token de autentificación válido

$ curl -X GET \
    https://network-api.onefootball.com/v1/integrations/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

Publicar contenido¶

Una vez que haya completado la configuración y tenga un token de autentificación, puede publicar un artículo en OneFootball.

Publicar un artículo¶

Con un token de autenticación válido, puede publicar un artículo enviando una solicitud POST al punto final de la entidad de la publicación.
Para hacerlo, use el siguiente ejemplo y haga lo siguiente

  • Reemplace TOKEN en el encabezado con su token de autentificación válido

  • Establezca todos los atributos del artículo como se muestra en el siguiente ejemplo.

$ curl -X POST \
    https://network-api.onefootball.com/v1/posts/ \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language": "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",
        "content": "Article content",
        "title": "Article title",
        "image_url": "https://your-blog.com/images/1.png",
        "image_width": 200,
        "image_height": 100,
        "draft": false
    }'

Descripción

external_id

obligatorio

El ID del artículo como se define en su sistema. Debe ser único (dentro de un idioma determinado) dentro de su propio sistema.

obligatorio

modified

obligatorio

La hora en que se actualizó el artículo por última vez. En caso de duda, utilice la hora actual.

content

obligatorio

title

obligatorio

El título del artículo. El título no puede ser una fila vacía.

image_url

opcional

Un campo opcional para la URL de la imagen destacada del artículo. Si se utiliza, se debe proporcionar image_width e image_height de la misma manera (el ancho y el alto de la imagen en píxeles).

draft

opcional

Un booleano opcional para ser utilizado con fines de prueba. Si se establece como true, el artículo no será visible para los usuarios de OneFootball. Si no se proporciona, el artículo estará disponible de forma predeterminada para los usuarios de OneFootball.

Actualizar y eliminar contenido¶

Una vez que su contenido esté en OneFootball, puede administrarlo actualizándolo y eliminándolo.
Para actualizar o eliminar un artículo en OneFootball, primero deberá obtener la ID de publicación de OneFootball para ese artículo.

Cómo obtener la ID de publicación de un artículo¶

Para obtener el ID de publicación de OneFootball para un artículo a fin de poder consultarlo en las operaciones de actualización o eliminación, siga el ejemplo que está abajo y haga lo siguiente:

  • Reemplace EXTERNAL_ID en la URL con la id del artículo en su sistema, la cual usted proporcionó al publicar el artículo. Esto debe ser codificado de ser necesario.

  • Reemplace TOKEN en el encabezado con su token de autentificación válido

$ curl -X GET \
    https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

import requests

headers = {
    'Authorization': 'Bearer TOKEN',
}

params = (
    ('external_id', 'EXTERNAL_ID'),
)

response = requests.get('https://network-api.onefootball.com/v1/posts/', headers=headers, params=params)

req, err := http.NewRequest("GET", "https://network-api.onefootball.com/v1/posts/?external_id=EXTERNAL_ID", nil)
if err != nil {

req.Header.Set("Authorization", "Bearer TOKEN")

resp, err := http.DefaultClient.Do(req)

defer resp.Body.Close()

Actualizar un artículo¶

Ocasionalmente, puede editar un artículo de su sitio. Para asegurarse de que este artículo se actualice en OneFootball, use el método PUT del punto final de la entidad de la publicación.
Para hacerlo, use el siguiente ejemplo y haga lo siguiente

  • Reemplace POST_ID en la URL con el ID de publicación de OneFootball del artículo que desea actualizar (consulte más arriba cómo obtenerla).

  • Reemplace TOKEN en el encabezado con su token de autentificación válido

  • Establezca todos los atributos del artículo como se muestra en el siguiente ejemplo. Tenga en cuenta que todos los atributos sobrescribirán los valores existentes.

$ curl -X PUT \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN' \
    -d '{
        "external_id": "ARTICLE_ID",
        "integration_id": INTEGRATION_ID,
        "source_url": "ARTICLE_URL",
        "language":  "en",
        "published": "2010-01-02T15:04:05Z",
        "modified": "2010-01-02T15:04:05Z",
        "content":  "Article content",
        "title":  "Article title",
        "image_url":  "https://your-blog.com/images/1.png",
        "image_width":  200,

Borrar un artículo¶

También puede eliminar artículos de OneFootball utilizando el método DELETE del punto final de la publicación.
Para hacerlo, use el siguiente ejemplo y haga lo siguiente

  • Reemplace POST_ID en la URL con el ID de publicación de OneFootball del artículo que desea eliminar (consulte más arriba cómo obtenerla).

  • Reemplace TOKEN en el encabezado con su token de autentificación válido

$ curl -X DELETE \
    https://network-api.onefootball.com/v1/posts/POST_ID \
    -H "Content-Type: application/json" \
    -H 'Authorization: Bearer TOKEN'

headers = {
    'Authorization': 'Bearer TOKEN',
}

response = requests.delete('https://network-api.onefootball.com/v1/posts/POST_ID', headers=headers)

 // handle err
